# Senior Fundamentals — Non-Coding Checklist

Condensed reminders of the 11 non-coding fundamentals of a senior FE engineer. Use as prompts to yourself — not every item every time, but the relevant 2–3 for the work you just finished.

Full reasoning and sources: `~/oktay-goals/senior/senior-fe-dev-non-coding-fundamentals.md`.

## Before Building (framing)

- [ ] **Problem Framing** — Which user/business problem does this solve? Did I question the proposed solution, not just implement it?
- [ ] **Saying No** — What happens if we do not do this now? Is scope creeping silently?
- [ ] **Cross-Functional** — Did I clarify edge cases / contracts with PM, Design, BE, QA, Data before assuming?
- [ ] **Estimation** — Did I share best / likely / worst case and surface risks early?

## While Building (delivery)

- [ ] **Technical Communication** — Does my PR description / commit / Slack note explain *why*, not just *what*? Did I include alternatives considered and rejected?
- [ ] **Glue Work** — Did I close one cross-team ambiguity (API contract, UX state, test scenario, doc gap)?
- [ ] **Self-Management** — Did I protect a focus block? Did I say no to a meeting / "quick question" that could be async?

## After Shipping (ownership)

- [ ] **Ownership** — What metric will this move? When and where will I check it (errors, analytics, user behavior)?
- [ ] **Spec Gaps** — Did I have to invent behavior because the spec was silent? Did I flag it back to the ticket / spec?
- [ ] **Feedback Loop** — Did I ask for one piece of feedback from manager / peer / reviewer?

## On Review (mentoring & feedback)

- [ ] **Mentoring** — Did my review comment teach the *principle*, not just request the change?
- [ ] **Sponsorship** — Is there a teammate who would grow from owning this next? Did I name them out loud?
- [ ] **Ego-less Feedback** — Am I critiquing the code, not the person? Did I add one positive observation alongside the constructive ones?

## Long-term (vision)

- [ ] **Tech Vision** — Does this change push the codebase toward or away from where it should be in 6–12 months?
- [ ] **Tech Debt Visibility** — If I took a shortcut, did I make the cost visible (ticket, ADR, PR note) rather than silent?

## Daily Cadence

A sustainable rhythm beats heroic days. Aim for:

- **Every day:** 1× Problem Framing, 1× Self-Management, 1× Ownership check
- **2–3× per week:** Tech Communication "why", Estimation with risk, one Cross-Functional sync
- **1× per week:** Deep mentoring feedback, one Sponsorship move, close one Glue ambiguity

Full daily checklist: `~/oktay-goals/senior/daily-non-coding-checklist-en.md`.

> Coding is "playing with the ball." Everything in this list is "off-ball work." Games are won off the ball.
